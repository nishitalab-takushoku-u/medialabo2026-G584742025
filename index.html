「Webコンテンツ作成」の成果

本テーマについて

HTML、CSS、JavaScriptの基本的な構文を学び、それらを組み合わせてWebページを作成する方法を習得する。また、ページの構造や見た目の設定、ユーザー操作に応じた処理や、外部サーバとの通信を実現する基礎的な技術を身に付ける。

プロジェクト課題について

今回のプロジェクト課題では、もともと実装されていたジャンル検索機能に加えて、自分でも「予算で絞り込めたら便利だ」と考え、予算検索機能を追加した。利用者が希望する価格帯のお店を探しやすくなるように、1000円ごとの範囲で予算を選択できるよう工夫した。また、誰が見ても何のサイトなのかがすぐに分かるように、ページ上部に「グルメスポット」というタイトルを配置し、検索エリアも見やすく整理した。さらに、ジャンルと予算の選択欄、検索ボタンをまとめて配置することで、「どこを指定して、どこをクリックすれば情報を得られるのか」が直感的に理解できるデザインを意識した。加えて、青色を基調とした配色やドット模様の装飾を取り入れ、見やすさだけでなく親しみやすいデザインになるよう工夫した。検索結果も表形式で表示することで、店名や予算、住所などの情報を比較しやすくし、利用者が目的のお店を探しやすいサイトを目指した。

プロジェクト課題で作成したページ project.html

数当てゲーム

足し算プログラムでは、テキストフィールドに入力された2つの数値を取得し、ボタンを押した際に計算を行い、その結果を画面上に表示する処理を実装した。実装を進める中で難しいと感じたのは、HTMLとJavaScriptの対応関係を理解しながら作業することであった。特に、どの要素をJavaScriptから参照したいのかを考え、そのためにHTML側で適切なidやname属性を設定する作業に苦労した。また、入力欄やボタン、結果表示用の要素がそれぞれどのような役割を持っているのかを整理しながら記述する必要があり、HTMLの構造を意識しながらプログラムを書く重要性を学ぶことができた。DOMを利用したイベント処理の基礎を理解する良い経験になった。

各授業回の学習事項

第1回 WWW と HTML

今回の授業では、基本的なHTMLの書き方について学んだ。HTMLを記述する際の基本的なルールをはじめ、見出しの作成方法、リンクの設定方法、画像の埋め込み方、説明リストの作成方法などを学習した。これらはいずれもWebページを作成する上で欠かせない要素であり、Webデザインの基礎として非常に重要な内容であった。今回学んだ知識を活用しながら、今後はより見やすく分かりやすいWebページの作成に取り組んでいきたい。

演習の成果: 演習1-4: HTMLのさまざまな要素

第2回 CSS による Web ページの装飾

今回の授業では、基本的なCSSの書き方について学んだ。CSSを用いることで、HTMLで記述された文章の色を変更したり、表や箇条書きを見やすく整えたりするなど、直感的に理解しやすいデザインを作成できることを学んだ。また、CSSを学習しながら試行錯誤を重ねることで、多くの人にとって見やすく使いやすいWebデザインを作成したいと感じた。今後は、利用者の視点を意識しながら、より分かりやすく魅力的なWebページの作成に取り組んでいきたい。

演習の成果 演習2-3:ウェブブラウザの紹介

第3回 JavaScript の基本

今回の授業では、JavaScriptについて学んだ。JavaScriptを用いることで、プログラムを通じてさまざまな複雑な機能を実現できることを理解した。特に、ツイートを検索・取得する機能や、GoogleマップをWebページに埋め込む機能は、Webデザインを学ぶ上で実用性が高く、活用の幅が広いと感じた。また、JavaとJavaScriptの違いについても学び、変数の型や比較演算子の使い方などに違いがあることを知った。さらに、for-of構文についても学習し、Javaの拡張for文に近い書き方であることを理解した。今後は、for-of構文を使いこなしながら、簡潔で見直しやすく、汎用性の高いJavaScriptのコードを書けるようになりたい。

演習の成果 演習3-6: オブジェクトの参照

第4回 DOM と JavaScript

今回の授業では、DOMとJavaScriptについて学んだ。DOMを利用することで、ボタンやテキストフィールドなどの要素を追加したり、ユーザーの操作に応じて処理を実行したりできることが分かった。特に、ユーザーが入力した内容やボタンのクリックをもとに動作を変えられる点が興味深かった。これまでのプログラミングでは、主にコンソールへの入力やコードの書き換えによって動作を確認していたが、今回は実際のWebページ上でユーザーが操作したときの動作を考えながらプログラムを作成した。そのため、より実用的なプログラム開発に近い内容を学ぶことができた。また、自分の作成したプログラムによって画面の表示が変化する様子を確認でき、DOMを使うことでWebページにさまざまな機能を追加できることが理解できた。

演習の成果 演習4-10: 足し算ページ

第5回 イベント処理

今回の授業では、イベント処理の基本的な書き方について学んだ。これまで学習してきたJavaScriptの基本的な文法に加え、DOMを利用することでWebページ上の要素を取得・編集したり、新しく作成したりできることを理解した。また、イベント処理を用いることで、ユーザーの操作に応じてページの内容を動的に変更できる点が興味深かった。演習では、DOMを操作して要素を追加する処理を実装したが、特に `beforeend` と `afterend` の使い分けに苦戦した。しかし、HTMLの構造を確認しながら、どの要素の内側に追加するのか、あるいは外側に追加するのかを意識して考えることで、正しく記述することができた。今回の学習を通して、JavaScriptによってWebページをより柔軟に操作できることを実感した。

演習の成果 演習5-7: DOMプログラミング(2)

第6回 axiosを用いた通信処理

今回の授業では、axiosを用いた通信処理について学んだ。Web上での通信は、ブラウザからサーバへデータを要求する処理と、サーバからブラウザへデータを返す処理の2つに分けて考えられることを理解した。また、JavaScriptでは通信を開始する処理だけでなく、通信が成功した場合の処理、失敗した場合の処理、そして通信終了後に実行する処理を記述する必要があることも学んだ。通信処理を利用することで外部のデータを取得でき、Webページに新たな機能を追加できることが分かり、興味深く感じた。

工夫点、苦心点、強調点、感想

今回のWebプログラミングの課題を通して、HTML、CSS、JavaScriptを利用したWebページ作成の流れを実際に体験することができた。特に最後のグルメ検索サイトの制作では、授業で学んだ内容を組み合わせながら、自分なりの工夫を加えて完成させることができたため、とても達成感があった。 最も苦労した点は、JavaScriptで取得したデータの参照方法である。グルメ情報は、店舗情報の中に予算やジャンルなどの情報がさらに入っている入れ子構造になっていた。そのため、自分が欲しい情報がどこに保存されているのかを理解し、「どの部分を参照すれば目的の情報を取り出せるのか」を考えることが非常に難しかった。実際にプログラムを書いていると、思った通りのデータが取得できずに悩むことも多かったが、コンソールを利用してデータの中身を確認しながら少しずつ原因を探し、解決していった。この経験から、プログラムを書く際には実際のデータ構造を理解することが非常に重要であると学んだ。 また、工夫した点として、先生から指定されていたジャンル検索だけでなく、予算による絞り込み機能を追加したことが挙げられる。授業で学んだジャンル検索の仕組みを応用し、同じ考え方で予算検索を実装した。飲食店を探す際にはジャンルだけでなく予算も重要な判断材料になるため、利用者にとってより便利なサイトになるよう意識した。実際には予算の表記に使われている文字の違いによって検索がうまく動かない問題も発生したが、コンソールを利用してデータを確認することで原因を特定し、修正することができた。この経験を通して、エラーが発生したときには焦らず原因を一つずつ調べることの大切さを実感した。 デザイン面でも工夫を行った。単に検索機能を実装するだけではなく、誰が見ても何を検索するサイトなのかが分かるように、「グルメスポット」というタイトルを大きく配置し、検索欄や検索ボタンを見やすい位置にまとめた。また、青色を基調とした配色やドット模様の装飾を取り入れることで、見やすさと親しみやすさの両立を目指した。Webサイトは機能だけでなく、利用者が迷わず操作できることも重要であるため、自分自身が利用者だったらどう感じるかを考えながら制作を進めた。 特に意識したのは利用者視点である。自分が飲食店を探すとしたら何の情報が欲しいのかを考え、ジャンルや予算だけでなく、駅からのアクセスや営業時間などの情報も一覧で確認できるようにした。本当は駅から徒歩何分かや喫煙可否、お店の雰囲気なども検索条件として追加したいと考えていたが、情報の取得方法や実装方法が分からず、今回は実現できなかった。しかし、そのような追加機能を考えられるようになったこと自体が、この授業を通して得られた成長だと思う。 さらに、コードを書く際には後から見返したときに理解しやすいよう、変数名や処理の流れを意識して記述するよう心掛けた。プログラムは動くだけでなく、自分や他の人が後から読んだときに理解しやすいことも大切であるため、できるだけ整理されたコードを書くよう努めた。 今回の学習を通して、普段何気なく利用しているWebサイトにも多くの工夫や技術が使われていることを実感した。以前は利用者としてサイトを見るだけだったが、今では「このデザインはどのように実装しているのだろう」「このボタンを押したらなぜ画面が変化するのだろう」といった視点で見るようになった。また、自分でも似たような機能を作れるのではないかと考えるようになり、Web技術への興味がさらに深まった。今後も授業で学んだ知識を活かしながら、より高度なUIや便利な機能の実装にも挑戦し、利用者にとって分かりやすく使いやすいWebサイトを制作できるようになりたい。
